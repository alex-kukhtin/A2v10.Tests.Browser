## Тесты А2v10

Система тестирования позволяет выполнять UI-тесты 
с использованием пакета [Selenium.WebDriver](https://www.selenium.dev).

## Предупреждение

Текущая система тестирования в ближайшее время будет заменена 
на пакеты тестирования с использованием Test Framework.


## Установка

1. Установите инструменты тестирования  из https://github.com/alex-kukhtin/A2v10.Tests.Browser/releases
2. Создайте новый проект Visual Studio (C# library .NET Framework 4.6.1).
3. Добавьте в References ссылку на библиотеку `C:\Program Files (x86)\a2v10\Tools\A2v10.Tests.Browser.dll`. 
Она нужна для того, чтобы работал Intellisense при вводе XAML-кода.
4. Создайте папку Tests. В этой папке нужно создать файл `Tests.Runner.config`. Формат файла описан ниже.
__Обратите внимание!__ Если этот проект будет располагаться в репозитории (я на это надеюсь), 
то обязательно добавьте этот файл в `.gitignore`, потому что там будут адреса и пароли.
5. В этом файле настраиваются хосты для тестирования, логины доступа и папки, в которых будут располагаться тесты.
6. Содайте внутри папки Tests подпапку с набором тестов. Например `01 Общие тесты`. Рекомендуется начинать имена папок с цифр, поскольку инструмент тестирования сортирует папки и тесты по имени.
7. Создаем ярлык на рабочем столе на файл `C:\Program Files (x86)\a2v10\Tools\A2v10.Tests.Runner.exe`
Настраиваем рабочий каталог (Working Directory) на папку `{path}\Tests` (`{path}` это путь, где лежит проект).
8. Пишем и запускаем тесты. Описание файлов тестов ниже.


## Формат .config файла.

Это стандартный файл конфигурации .NET.
```xml
<?xml version="1.0" encoding="utf-8" ?>
<configuration>
  <configSections>
    <section name="hosts" allowDefinition="Everywhere" type="A2v10.Tests.Browser.HostsSection, A2v10.Tests.Browser"/>
  </configSections>
  <hosts>
    <host name="a2v10.com">
      <source name="Первый набор тестов" directory="01 Общие тесты" url="https://a2v10.com" login="login_here" password="password_hrere" company=""/>
      <source name="Второй набор тестов" directory="02 Еще тесты" url="https://a2v10.com" login="login_here" password="password_hrere" company=""/>
    </host>
  </hosts>
</configuration>
```

Секция `configSections` стандартная и разрешает создание секций `hosts`.

Секция `hosts` включает описание одного или нескольких хостов, которые мы будем тестировать.

Каждая секция `host` имеет имя и включает один или несколько наборов тестов.

Каждый набор тестов имеет имя, привязку к папке с тестами, URL приложения, которое будет тестироваться, имя и пароль для входа и, возможно компанию внутри приложения 
(нужно в режиме multicompany).

## Формат файлов тестов

Файлы тестов должны иметь расширение `.feature.xaml`. 

Каждый тест тестирует одну возможность (Feature), которая включает несколько сценариев (Scenario).

Каждый сценарий состоит из одного или нескольких шагов (Steps). Некоторые шаги (например Dialog) 
могут октрывать собственную область видимости (Scope). Все, что описывается внутри шага относится к 
этой области видимости. Внутри области видимости можно выполнять действия (Actions) 
или проверять состояния (States)

### Шаги (Steps)

* **Navigate**  - навигация на заданный url. Url задается атрибутом - *Url*.
* **Dialog**  - Открывает новую область видимости с текущим диалогом. Все шаги внутри будут 
относиться к диалогу. Проверка диалога - свойства *Title*, *TestId* (задается в Xaml).
* **Confirm** - То же самое, что и диалог.
* **Control** - Открывает новую область видимости с выбранным элементом. Все шаги внутри - *ElementStep* 
* относятся к этому элементу управления.
* **TreeView** - Открывает новую область видимости с деревом.
* **NewWindow**  - Открывает новую область видимости с текущим открытым окном. Этот шаг будет 
всегда после выполнения команды, которая открывает что-то в новом окне.
Все шаги внутри будут 
относиться к этому окну. Проверка окта - свойства *Title* и *Url*. При выходе из области видимости, 
окно закроется автоматически.
* **DataGrid**  - Открывает новую область видимости *DataGrid*. Внутри нее досупны специфические действия.

## Действия (Actions)

* **ClickButton** - нажать кнопку. Кнопку можно задать текстом (атрибут *Text*) или иконкой (атрибут *Icon*).
* **ClickMenuItem**  - нажать элемент меню. Элемент можно задать текстом (атрибут *Text*).
* **ClickLink** - нажать ссылку. Ссылка задается атрибутом *Url* или текстом (атрибут *Text*).
* **ClickListItem** - нажать элемент списка. Элемент задается атрибутом *Header*.
* **ClickTabBarButton** - нажать на ссылку в *TabBar*. Задается атрибутом *Text*.
* **ClickAddOn** - нажать ссылку AddOn. Задается иконкой (атрибут *Icon*).
* **Escape**  - Нажать клавишу *Escape*.
* **Enter**  - Нажимает *Enter* на элементе.
* **TypeText**  - Вводит текст в поле элемента. Задается свойством *Text*.
* **EnterText**  - Вводит текст в поле элемента и нажимает клавишу *Enter*. Задается свойством *Text*.
* **ClickCaret**  - Нажимает на "клювик" селектора.
* **SelectorPane**  - Открывает новую область видимости с панелью селектора. Все шаги внутри будут 
относиться к этой панели.
* **Select**  - Выбирает элемент в панели селектора. Задается свойством *Text*.
* **WaitForComplete** - Ожидает завершения действия.

## Состояния (States)

* **Valid**  - Проверяет, что элемент корректен.
* **Invalid**  - Проверяет, что элемент в состоянии ошибки. Ошибку можно проверить через свойство *Message*.
* **EnsureEmpty**  - Проверяет, что элемент пуст.
* **EnsureText**  - Проверяет, что элемент содержит текст. . Задается свойством *Text*.
* **Disabled**  - Проверяет, что элемент запрещен.
